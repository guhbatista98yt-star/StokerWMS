- logs de integração
- logs de erro

Quero que os logs sejam:

- sem stack trace perdido
- sem erro engolido

PADRÃO DE LOG DESEJADO
Sempre que fizer sentido, registrar:
- timestamp
- nível (info, warn, error, debug)
- módulo
- ação/evento

- id da entidade relevante (pedido, work unit, item, operador, etc.)
- request id / correlation id
- resultado da operação
- duração da operação
- mensagem clara
- stack trace quando houver erro
- payload sanitizado quando necessário

IMPORTANTE:
- não registrar senha, token sensível, segredo ou dados indevidos
- não deixar log genérico tipo “deu erro”
- não esconder erro real atrás de mensagem vaga
- não encher o log de ruído inútil
- diferenciar log operacional de log técnico

Também quero:
- padronização do formato dos logs
- melhoria de nomenclatura
- captura de erros não tratados
- melhor rastreabilidade de problemas em produção
- revisão de logs redundantes ou inúteis
- log de falha de integração com contexto
- log de erro de regra de negócio com contexto
- log de timeout/retry/falha de rede
- log de falha de persistência
- log de falha em transição de status
- log de inconsistência detectada
