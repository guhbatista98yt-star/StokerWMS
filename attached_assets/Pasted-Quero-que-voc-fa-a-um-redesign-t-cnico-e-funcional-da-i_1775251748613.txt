Quero que você faça um redesign técnico e funcional da interface da minha aplicação WMS, usando como referência visual os layouts que anexei, mas adaptando isso para um sistema operacional real, leve, rápido e consistente.

IMPORTANTE:
Não quero uma cópia literal das imagens.
Quero traduzir a linguagem visual delas para um WMS utilizável em produção.

OBJETIVO GERAL
Modernizar a interface do sistema para uma identidade visual premium, limpa e consistente, com:
- menu lateral elegante
- dark mode e light mode coerentes
- cards, tabelas e formulários padronizados
- boa experiência para desktop, tablet e mobile
- foco em módulos operacionais de WMS
- layout bonito, mas principalmente funcional

ANTES DE IMPLEMENTAR
1. Analise a stack atual da aplicação.
2. Identifique:
   - framework frontend
   - biblioteca de componentes, se existir
   - sistema de estilos atual
   - estrutura de layout
   - padrões visuais atuais
   - componentes reutilizáveis já existentes
3. Preserve a regra de negócio atual.
4. Não altere fluxos operacionais sem necessidade.
5. Faça a refatoração visual de forma organizada, sem quebrar funcionalidades existentes.
6. Se a aplicação já usar React/Tailwind/shadcn, siga esse padrão.
7. Se usar outra stack, adapte a arquitetura equivalente, mantendo os mesmos princípios de design system e theming.

DIREÇÃO VISUAL
Usar como inspiração os layouts enviados:
- azul profundo como cor primária
- tema dark elegante com preto azulado
- tema light com fundo off-white/cinza claro
- bordas suaves
- cards bem construídos
- menu lateral visualmente forte
- interface limpa e moderna

PORÉM:
- sem excesso de efeitos
- sem visual de portfólio
- sem grandes áreas vazias desnecessárias
- sem composições inclinadas/diagonais no uso real
- sem excesso de blur
- sem transparências que prejudiquem leitura
- sem animações pesadas

O sistema precisa ter aparência premium, mas com foco operacional.

LIGHT MODE:
- fundo principal: off-white / cinza muito claro

- manter contraste e elegância

LAYOUT PRINCIPAL
Quero uma arquitetura de layout consistente para o sistema inteiro.

1) APP SHELL
Criar/refatorar um shell principal com:
- sidebar lateral
- header/topbar
- área de conteúdo principal
- breadcrumbs quando fizer sentido
- espaço para filtros e ações globais
- suporte a desktop e mobile

2) SIDEBAR
Quero um menu lateral inspirado nas referências, mas funcional:
- fixo no desktop
- colapsável
- com largura expandida e largura compacta
- ícones consistentes
- labels claros
- item ativo muito bem definido
- hover discreto
- submenu elegante
- versão mobile em drawer
- boa legibilidade em dark e light mode

Sugestão de medidas:
- expanded: 260px a 280px
- collapsed: 72px a 84px

3) HEADER / TOPBAR
Criar cabeçalho limpo com:
- título da página

- ações principais
- busca, quando fizer sentido
- toggle de tema

MÓDULOS WMS – PRIORIDADE FUNCIONAL
Quero atenção especial aos módulos operacionais:
- separação
- conferência
- recebimento
- estoque
- inventário
- endereçamento
- movimentação
- romaneio

Nesses módulos, a UI deve priorizar:
- velocidade
- legibilidade
- status claros
- áreas de clique/toque adequadas
- baixa fricção
- leitura rápida de item atual
- ação principal da tela evidente
- menos decoração e mais eficiência

PADRÃO VISUAL PARA MÓDULOS OPERACIONAIS
Crie um padrão específico para telas operacionais, diferente do dashboard gerencial:
- mais densidade de informação
- menos área vazia
- cards mais compactos
- tabelas/listas mais objetivas
- contraste forte
- feedback visual rápido
- indicadores visuais de status do item/pedido atual
